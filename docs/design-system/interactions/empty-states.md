# 25. EMPTY STATES

An empty state is not an absence of design.

Every major surface must define:

- loading state
- empty state
- error state
- success state

---

## Empty State Goals

Empty states should:

- explain why content is absent
- guide the user toward the next action
- reduce confusion
- preserve layout stability
- maintain visual hierarchy

---

## Structure

A proper empty state may include:

- icon or illustration
- title
- supporting description
- primary action
- optional secondary action

---

## Rules

Avoid generic placeholders such as:

- "No data"
- "Nothing here"

Prefer contextual guidance.

Good:

- "No projects yet. Create your first project to begin."

Bad:

- "Empty."

---

## Layout

Empty states should:

- remain centered within their container
- preserve spacing rhythm
- avoid excessive decoration
- support both light and dark mode
