# COMPONENT ENGINEERING RULES

Components must be:

- reusable
- composable
- predictable
- accessible
- documented

Avoid:

- giant prop APIs
- deeply nested logic
- visual inconsistency
- duplicated styles

Prefer:

- composition
- variants
- token-driven styling
- modular architecture

## CONSISTENCY OVER NOVELTY

Consistency scales.
Novelty decays.

Do not reinvent:

- spacing
- typography
- shadows
- animation curves
- interaction patterns

The system should feel unified across all components.
